```
┌─────────────────────────────────────────┐

├─────────────────────────────────────────┤
│  第 3 层: Agent 能力 (核心差异)          │
│  - 工具数量与类型                        │
│  - 自主决策能力                          │
│  - 多步骤任务执行                        │
├─────────────────────────────────────────┤
│  第 2 层: 上下文管理 (关键差异)          │

│  - RAG / 代码索引                        │
├─────────────────────────────────────────┤

│  - 终端命令执行                          │
└─────────────────────────────────────────┘
```

## 🔍 各层面的核心差异

### 1️⃣ 架构差异 (第 1-2 层)

| 方面 | 不同选择 |
|------|----------|
| **代码解析** | Tree-sitter vs LSP vs 简单文本 |
| **上下文构建** | 全文发送 vs 智能检索 vs 用户手动选择 |
| **状态管理** | 有状态对话 vs 无状态请求 |
| **执行环境** | 本地沙箱 vs 直接系统调用 |

**例子**：
- **Aider**: 用 Tree-sitter 解析代码结构，智能选择相关文件
- **Cline**: 使用 VS Code 的 LSP 获取代码语义
- **Cursor**: 自研代码索引系统，支持整个代码库的语义搜索

---

### 2️⃣ Agent 能力差异 (第 3 层) - **最关键**

这是真正的分水岭：

| 能力 | Aider | Cline | Claude Code | Cursor |
|------|-------|-------|-------------|--------|
| **工具数量** | 少而精 | 中等 | 丰富 | 丰富 |
| **浏览器自动化** | ❌ | ✅ | ❌ | ✅ |
| **自主规划** | 有限 | 强 | 强 | 强 |
| **多文件协调** | ✅ | ✅ | ✅ | ✅ |
| **MCP 扩展** | ❌ | ✅ | ❌ | ❌ |

**核心差异 = Tool Use + Planning 能力**

```python
# 简单工具调用 (Aider 风格)
tools = ["read_file", "write_file", "run_command"]

# 复杂 Agent (Cline/Claude Code 风格)
tools = [
    "analyze_codebase",      # 代码分析
    "search_files",          # 文件搜索
    "read_file", 
    "write_file",
    "run_terminal_command",
    "use_browser",           # 浏览器操作
    "ask_followup",          # 主动提问
    "complete_task"          # 任务完成标记
]
```

---

### 3️⃣ Prompt 工程差异 (贯穿各层)

Prompt 设计决定了 AI 的行为模式：

| 工具 | Prompt 特点 |
|------|-------------|
| **Aider** | 简洁直接，专注代码编辑 |
| **Cline** | 结构化，支持多步骤推理 |
| **Claude Code** | 详细的系统提示，强调安全和协作 |
| **Cursor** | 商业化调优，平衡创意和准确 |

**关键 Prompt 技巧**：
- **System Prompt**: 定义 AI 角色和能力边界
- **Tool Description**: 影响工具选择质量
- **Few-shot Examples**: 引导输出格式
- **Context Assembly**: 如何组织代码上下文

---

### 4️⃣ Skill / 工作流差异 (第 3-4 层)

| 概念 | 说明 |
|------|------|
| **Skill** | 预定义的任务模板 (如"创建 React 组件") |
| **Workflow** | 多步骤的协作流程 |

**例子对比**：

**Aider** - 无预设 Skill，纯对话驱动
```
用户: 添加一个用户认证功能
Aider: 好的，我需要修改哪些文件？
```

**Cline** - 支持 MCP 扩展自定义工具
```
用户: 添加一个工具来获取 Jira 工单
Cline: 创建 MCP server → 安装 → 使用
```

**Cursor** - 内置多种 Skill
- `/generate` - 代码生成
- `/edit` - 智能编辑
- `/chat` - 自由对话
- `/debug` - 错误诊断

## 🎯 总结：差异的核心

```
最主要的差异排序：

1. Agent 架构 (Tool Use + Planning)
   └── 这是"智能"程度的关键
   
2. 上下文管理策略
   └── 决定能否处理大项目
   
3. 产品形态和交互设计
   └── 影响用户体验和工作流
   
4. Prompt 工程
   └── 影响输出质量和稳定性
   
5. Skill/工作流预设
   └── 提升特定场景效率
```
