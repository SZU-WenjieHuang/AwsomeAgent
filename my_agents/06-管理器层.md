### 三、SessionManager 详解

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                           SessionManager 详解                                   │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────┐
│ 1. 核心职责                                                                     │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   • 会话元数据管理（创建、查询、更新、删除）                                    │
│   • SQLite 持久化存储                                                          │
│   • 内存缓存 + 索引加速查询                                                    │
│   • 用户隔离（每个用户只能看到自己的会话）                                      │
│                                                                                 │
│   注意：SessionManager 只管理会话元数据，不存储消息内容                         │
│         消息内容由 LangGraph Checkpointer 管理                                  │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────┐
│ 2. 数据模型                                                                     │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   Session 模型                                                                 │
│   ┌─────────────────────────────────────────────────────────────────────────┐  │
│   │                                                                         │  │
│   │   class Session:                                                        │  │
│   │       id: str              # UUID，唯一标识                              │  │
│   │       user_id: str         # 所属用户                                   │  │
│   │       model: str           # 使用的模型                                 │  │
│   │       status: str          # 状态：active / archived / deleted          │  │
│   │       created_at: str      # 创建时间 (ISO 格式)                        │  │
│   │       updated_at: str      # 更新时间 (ISO 格式)                        │  │
│   │                                                                         │  │
│   └─────────────────────────────────────────────────────────────────────────┘  │
│                                                                                 │
│   SQLite 表结构                                                                │
│   ┌─────────────────────────────────────────────────────────────────────────┐  │
│   │                                                                         │  │
│   │   CREATE TABLE sessions (                                               │  │
│   │       id TEXT PRIMARY KEY,                                              │  │
│   │       user_id TEXT NOT NULL,                                            │  │
│   │       model TEXT NOT NULL,                                              │  │
│   │       status TEXT NOT NULL DEFAULT 'active',                            │  │
│   │       created_at TEXT NOT NULL,                                         │  │
│   │       updated_at TEXT NOT NULL                                          │  │
│   │   )                                                                     │  │
│   │                                                                         │  │
│   │   -- 索引                                                                │  │
│   │   CREATE INDEX idx_sessions_user_id ON sessions(user_id)               │  │
│   │   CREATE INDEX idx_sessions_status ON sessions(status)                  │  │
│   │   CREATE INDEX idx_sessions_updated_at ON sessions(updated_at)          │  │
│   │                                                                         │  │
│   └─────────────────────────────────────────────────────────────────────────┘  │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────┐
│ 3. 内存缓存 + 索引设计                                                         │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   ┌─────────────────────────────────────────────────────────────────────────┐  │
│   │                                                                         │  │
│   │   class SessionManager:                                                 │  │
│   │       _cache: Dict[str, Session]        # session_id → Session          │  │
│   │       _user_index: Dict[str, List[str]] # user_id → [session_ids]       │  │
│   │       _status_index: Dict[str, List[str]] # status → [session_ids]      │  │
│   │                                                                         │  │
│   └─────────────────────────────────────────────────────────────────────────┘  │
│                                                                                 │
│   查询示例                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────────┐  │
│   │                                                                         │  │
│   │   # 查询用户的所有会话                                                   │  │
│   │   user_sessions = _user_index.get("user_a", [])                        │  │
│   │               → ["session_1", "session_2", ...]                         │  │
│   │                                                                         │  │
│   │   # 查询所有活跃会话                                                     │  │
│   │   active_sessions = _status_index.get("active", [])                    │  │
│   │                                                                         │  │
│   │   # 组合查询：用户 + 状态                                                │  │
│   │   user_active = set(_user_index[user_id]) & set(_status_index["active"])│  │
│   │                                                                         │  │
│   └─────────────────────────────────────────────────────────────────────────┘  │
│                                                                                 │
│   优势：                                                                       │
│   • 读取操作完全在内存中，无需访问数据库                                        │
│   • 写入操作同时更新缓存和数据库，保持一致性                                    │
│   • 索引支持 O(1) 复杂度的用户/状态过滤                                        │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────┐
│ 4. 核心方法                                                                     │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   async initialize()                                                           │
│       创建表 + 加载已有会话到缓存                                              │
│                                                                                 │
│   async create_session(user_id, model) → Session                               │
│       创建新会话，写入 SQLite + 更新缓存                                       │
│                                                                                 │
│   async get_session(session_id) → Optional[Session]                            │
│       从缓存获取会话（O(1)）                                                   │
│                                                                                 │
│   async list_sessions(user_id=None, status=None) → List[Session]               │
│       使用索引快速过滤，按 updated_at 倒序排列                                 │
│                                                                                 │
│   async update_session(session_id, status) → Optional[Session]                 │
│       更新会话状态，同步更新数据库和缓存                                        │
│                                                                                 │
│   async delete_session(session_id) → bool                                      │
│       删除会话，同时清理缓存和索引                                             │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────┐
│ 5. 与 Checkpointer 的关系                                                      │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   ┌─────────────────────────────────────────────────────────────────────────┐  │
│   │                                                                         │  │
│   │   SessionManager                    CheckpointerManager                 │  │
│   │   ┌─────────────────────┐          ┌─────────────────────┐             │  │
│   │   │ 管理会话元数据       │          │ 管理会话状态         │             │  │
│   │   │                     │          │                     │             │  │
│   │   │ • session_id        │          │ • messages          │             │  │
│   │   │ • user_id           │          │ • agent_state       │             │  │
│   │   │ • model             │          │ • tool_history      │             │  │
│   │   │ • status            │          │                     │             │  │
│   │   │ • created_at        │          │ 由 LangGraph 管理    │             │  │
│   │   │ • updated_at        │          │                     │             │  │
│   │   └─────────────────────┘          └─────────────────────┘             │  │
│   │           │                                  │                         │  │
│   │           │      session_id 作为关联键       │                         │  │
│   │           └──────────────────────────────────┘                         │  │
│   │                                                                         │  │
│   │   关系：                                                                 │  │
│   │   • SessionManager 创建会话时生成 session_id                            │  │
│   │   • TaskClawAgent 使用 session_id 作为 thread_id 传给 Checkpointer     │  │
│   │   • Checkpointer 存储该会话的所有消息和状态                             │  │
│   │                                                                         │  │
│   └─────────────────────────────────────────────────────────────────────────┘  │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

协作场景示例
─────────────────────────────────────────────────────────────────────────────────

场景 1: 创建新会话
─────────────────
1. AgentManager.get_or_create_agent(user_id)
   └─→ ConfigManager.load() 读取用户配置
   └─→ McpManager.get_enabled_servers() 获取 MCP 配置
   └─→ SkillManager.get_enabled_skill_paths() 获取 Skill 路径

2. AgentManager.create_session(user_id)
   └─→ SessionManager.create_session(user_id, model)
       └─→ 写入 sessions.db

场景 2: 热更新 MCP 工具
───────────────────────
1. 用户调用 POST /api/mcp/reload
2. McpManager 更新 config.yaml
3. AgentManager.reload_mcp_tools(user_id)
   └─→ TaskClawAgent.reload_mcp_tools()
       └─→ McpToolLoader.connect_and_load_tools()
       └─→ 重建 DeepAgent

场景 3: 创建新 Skill
───────────────────
1. 用户调用 POST /api/skills/create
2. SkillManager.create_skill()

### 总结

| 问题 | 回答 |
|------|------|
| **Manager 在哪里调用？** | AgentManager 是核心调用者，协调所有 Manager；ConfigManager 在 TaskClawAgent 初始化时使用；SessionManager 管理 API 路由中的会话；McpManager/SkillManager 通过 AgentManager 暴露给 API |
| **SessionManager 存什么？** | 只存元数据（id, user_id, model, status, 时间戳）；消息和状态由 LangGraph Checkpointer 管理；两者共享同一个 SQLite 文件，通过 session_id = thread_id 关联 |
| **多 Agent 改造需要变动吗？** | 需要中等程度改造：SessionManager 增加字段、McpManager/SkillManager 按 agent_type 分组、ConfigManager 支持按 agent_type 读取配置；CheckpointerManager 无需改造 |
