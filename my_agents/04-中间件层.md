## Middleware 层完整分析

### 一、AgentMiddleware 基类架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    AgentMiddleware 基类（来自 langchain.agents.middleware）  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  生命周期 Hooks（按执行顺序）：                                              │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 1. before_agent / abefore_agent                                     │   │
│  │    触发时机：Agent 开始执行前                                        │   │
│  │    用途：初始化、数据准备、状态检查                                   │   │
│  │    返回值：Dict[str, Any] → 合并到 state                            │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                              ↓                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 2. wrap_model_call / awrap_model_call                               │   │
│  │    触发时机：每次 LLM 调用前后                                       │   │
│  │    用途：修改请求/响应、token 统计、请求日志                          │   │
│  │    参数：ModelRequest → handler → ModelResponse                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                              ↓                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 3. wrap_tool_call / awrap_tool_call                                 │   │
│  │    触发时机：每次工具调用前后                                        │   │
│  │    用途：错误处理、参数校验、调用日志                                 │   │
│  │    参数：ToolCallRequest → handler → ToolMessage                    │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                              ↓                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 4. after_agent / aafter_agent                                       │   │
│  │    触发时机：Agent 执行完成后                                        │   │
│  │    用途：清理、状态持久化、后处理                                     │   │
│  │    返回值：Dict[str, Any] → 合并到 state                            │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  辅助方法：                                                                 │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ modify_request(request: ModelRequest) → ModelRequest               │   │
│  │    修改 LLM 请求（如注入 system prompt）                             │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 二、TaskClawAgent 中 Middleware 的使用方式

```python
# taskclaw_agent.py 第 494-538 行

# 创建 middlewares
self.user_data_middleware = UserDataMiddleware(
    backend_root_dir=self.backend_root_dir,
    user_id=self.user_id,
)
self.memory_middleware = MemoryMiddleware(
    backend_root_dir=self.backend_root_dir,
    user_id=self.user_id,
)
self.tool_error_middleware = ToolErrorHandlingMiddleware()

# 传递给 create_deep_agent
self.agent = create_deep_agent(
    model=llm,
    system_prompt=TASK_AGENT_WORKFLOW_INSTRUCTIONS,
    tools=all_tools,
    skills=self._skills,
    checkpointer=checkpointer,
    backend=backend,
    middleware=[
        self.tool_error_middleware,  # ← 第1个执行（最外层）
        self.user_data_middleware,   # ← 第2个执行
        self.memory_middleware,      # ← 第3个执行（最内层）
    ],
    debug=False,
)
```

### 三、各 Middleware 详细分析

#### 1. ToolErrorHandlingMiddleware

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    ToolErrorHandlingMiddleware                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  文件位置：middlewares/tool_error_middleware.py                             │
│  核心职责：捕获工具执行异常，返回结构化错误 ToolMessage                       │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 实现的 Hooks                                                         │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │ ✅ wrap_tool_call()    同步工具调用包装                              │   │
│  │ ✅ awrap_tool_call()   异步工具调用包装                              │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  工作原理：                                                                 │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │   try:                                                              │   │
│  │       return handler(request)  # 执行工具                           │   │
│  │   except GraphBubbleUp:                                             │   │
│  │       raise  # LangGraph interrupt 必须传播                         │   │
│  │   except Exception as e:                                            │   │
│  │       # 构造友好的错误消息                                           │   │
│  │       return ToolMessage(                                           │   │
│  │           content=self._format_error_message(tool_name, e, args),   │   │
│  │           tool_call_id=tool_call_id,                                │   │
│  │           status="error",                                           │   │
│  │       )                                                             │   │
│  │                                                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  错误消息格式：                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ ❌ 工具调用失败: search_poi                                          │   │
│  │                                                                     │   │
│  │ **错误类型**: ValueError                                             │   │
│  │ **错误信息**: Invalid POI ID format                                 │   │
│  │ **调用参数**: {"poi_id": "B000A7J3GW"}                              │   │
│  │                                                                     │   │
│  │ **可能原因**:                                                        │   │
│  │ - 参数格式不正确或缺少必需参数                                        │   │
│  │ - 参数值类型不匹配                                                   │   │
│  │                                                                     │   │
│  │ **建议**:                                                            │   │
│  │ - 检查工具的参数定义，确保参数格式正确                                 │   │
│  │ - 如果参数来自其他工具的输出，检查输出格式是否匹配                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  设计目的：                                                                 │
│  - 防止工具异常导致 Agent 循环终止                                          │
│  - 让 LLM 理解错误原因并自我修正                                            │
│  - 区分参数错误 vs 工具内部错误                                             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 五、自定义 Middleware 模板

```python
from typing import Any, Dict, Optional, Callable, Awaitable
from langchain.agents.middleware import AgentMiddleware
from langchain.agents.middleware.types import (
    AgentState,
    ModelRequest,
    ModelResponse,
    ToolCallRequest,
)
from langchain_core.messages import ToolMessage
from langgraph.runtime import Runtime


class CustomMiddleware(AgentMiddleware[AgentState]):
    """自定义 Middleware 模板"""
    
    # 可选：定义状态 Schema
    state_schema = AgentState
    
    # ============== Agent 生命周期 Hooks ==============
    
    def before_agent(
        self, 
        state: AgentState, 
        runtime: Runtime
    ) -> Optional[Dict[str, Any]]:
        """Agent 执行前调用
        
        返回值会合并到 state 中
        """
        return None
    
    def after_agent(
        self, 
        state: AgentState, 
        runtime: Runtime
    ) -> Optional[Dict[str, Any]]:
        """Agent 执行后调用
        
        返回值会合并到 state 中
        """
        return None
    
    # ============== LLM 调用 Hooks ==============
    
    def modify_request(self, request: ModelRequest) -> ModelRequest:
        """修改 LLM 请求（如注入 system prompt）
        
        在 wrap_model_call 之前调用
        """
        return request
    
    def wrap_model_call(
        self,
        request: ModelRequest,
        handler: Callable[[ModelRequest], ModelResponse],
    ) -> ModelResponse:
        """包装 LLM 调用
        
        可以：
        - 修改请求参数
        - 记录请求日志
        - 统计 token 使用
        - 重试失败请求
        """
        # 修改请求（可选）
        modified_request = self.modify_request(request)
        
        # 调用 LLM
        response = handler(modified_request)
        
        # 后处理（可选）
        # ...
        
        return response
    
    # ============== Tool 调用 Hooks ==============
    
    def wrap_tool_call(
        self,
        request: ToolCallRequest,
        handler: Callable[[ToolCallRequest], ToolMessage],
    ) -> ToolMessage:
        """包装工具调用
        
        可以：
        - 参数校验
        - 权限检查
        - 错误处理
        - 调用日志
        """
        try:
            return handler(request)
        except Exception as e:
            # 错误处理示例
            return ToolMessage(
                content=f"工具调用失败: {e}",
                tool_call_id=request.tool_call.get("id", ""),
                status="error",
            )
    
    # ============== 异步版本（可选） ==============
    
    async def abefore_agent(self, state, runtime):
        return self.before_agent(state, runtime)
    
    async def aafter_agent(self, state, runtime):
        return self.after_agent(state, runtime)
    
    async def awrap_model_call(self, request, handler):
        modified_request = self.modify_request(request)
        response = await handler(modified_request)
        return response
    
    async def awrap_tool_call(self, request, handler):
        try:
            return await handler(request)
        except Exception as e:
            return ToolMessage(
                content=f"工具调用失败: {e}",
                tool_call_id=request.tool_call.get("id", ""),
                status="error",
            )
```

## Middleware 本质

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     Middleware = 生命周期 Hooks 集合                         │
└─────────────────────────────────────────────────────────────────────────────┘

Agent 执行流程                        Hook 触发点
─────────────────────────────────────────────────────────────────────────────

  用户请求进入
       │
       ▼
┌─────────────────┐
│  before_agent   │  ← Hook 1: 执行前初始化
│                 │     例：创建目录、加载记忆、重置状态
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ wrap_model_call │  ← Hook 2: LLM 调用前后
│                 │     例：注入 prompt、token 统计、请求日志
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ wrap_tool_call  │  ← Hook 3: 工具调用前后
│                 │     例：错误处理、权限检查、调用日志
└────────┬────────┘
         │
    (循环执行 LLM + Tool)
         │
         ▼
┌─────────────────┐
│  after_agent    │  ← Hook 4: 执行后清理
│                 │     例：保存状态、更新记忆、资源释放
└─────────────────┘
         │
         ▼
     返回结果
```

## 简化理解

```python
# 没有 Middleware 的执行流程
def run_agent(state):
    # 直接执行
    response = llm.invoke(state)
    while need_tool(response):
        result = tool.invoke(response.tool_call)
        response = llm.invoke(result)
    return response

# 有 Middleware 的执行流程
def run_agent_with_middleware(state, middlewares):
    # Hook 1: before_agent
    for m in middlewares:
        state.update(m.before_agent(state) or {})
    
    while not done:
        # Hook 2: wrap_model_call（洋葱模型，从外到内）
        handler = lambda req: llm.invoke(req)
        for m in reversed(middlewares):
            handler = lambda req, h=handler, m=m: m.wrap_model_call(req, h)
        response = handler(state)
        
        if need_tool(response):
            # Hook 3: wrap_tool_call（洋葱模型）
            handler = lambda req: tool.invoke(req)
            for m in reversed(middlewares):
                handler = lambda req, h=handler, m=m: m.wrap_tool_call(req, h)
            result = handler(tool_request)
    
    # Hook 4: after_agent
    for m in middlewares:
        state.update(m.after_agent(state) or {})
    
    return response
```

## 一句话总结

| 概念 | 本质 |
|------|------|
| Middleware | 一组生命周期 Hook 函数的容器 |
| Hook | 在特定时机触发的回调函数 |
| 洋葱模型 | 多个 Middleware 嵌套执行，请求从外到内，响应从内到外 |

**所以 Middleware 层就是：通过 Hook 机制，在不修改 Agent 核心逻辑的前提下，插入横切关注点（日志、错误处理、记忆管理等）。**
