## MoE vs MoA：从原理到工程的完整对比

---

### 一、MoE（Mixture of Experts）是什么

MoE 是一种**模型内部的架构设计**，起源于 1991 年，在 LLM 时代被 Google 的 Switch Transformer、Mistral 的 Mixtral 等大规模应用。

**核心思路**：一个模型里有多个"专家子网络"，每次推理时只激活其中一部分。

**数学形式**（论文里给出的）：

$$y_i = \sum_{j=1}^{n} G_{i,j}(x_i) \cdot E_{i,j}(x_i) + x_i$$

用人话翻译：

```
输出 = 门控网络打分 × 专家网络输出 的加权求和 + 残差连接
```

**架构图**：

```
输入 token
    │
    ├─→ 门控网络（Gating Network）
    │       │ 计算每个专家的权重
    │       ↓
    │   [0.7, 0.1, 0.0, 0.2, ...]  ← 只激活 top-k 个专家
    │
    ├─→ Expert 1（FFN 子网络）─→ × 0.7
    ├─→ Expert 2（FFN 子网络）─→ × 0.1
    ├─→ Expert 3（FFN 子网络）─→ × 0.0（未激活）
    ├─→ Expert 4（FFN 子网络）─→ × 0.2
    │
    └─→ 加权求和 → 输出
```

**关键特性**：
- **稀疏激活**：每次只激活 top-k 个专家（通常 k=2），参数量大但计算量小
- **参数量 vs 计算量分离**：Mixtral 8x7B 有 46B 参数，但每次推理只用 12B
- **在模型内部**：用户/开发者感知不到专家的存在，是模型架构层的事
- **需要训练**：门控网络和专家网络都要联合训练

**代表模型**：
- Mixtral 8x7B / 8x22B（Mistral）
- GPT-4（据传是 MoE 架构）
- Switch Transformer（Google）
- DeepSeek-V2/V3（也是 MoE）

### 二、MoA（Mixture of Agents）是什么

MoA 是一种**系统级的多模型协作架构**，是论文 arXiv:2406.04692 提出的。

**核心思路**：多个完整的 LLM 分层协作，前一层的输出作为后一层的输入辅助信息。

**数学形式**（论文原文）：

$$y_i = \bigoplus_{j=1}^{n}[A_{i,j}(x_i)] + x_1, \quad x_{i+1} = y_i$$

用人话翻译：

```
第 i 层输出 = 聚合(所有 Agent 的回答) + 原始问题
下一层输入 = 上一层输出
```

**架构图**：

```
用户问题
    │
    ├─────────────────────────────────┐
    │         Layer 1（并行）          │
    │  LLM-A  LLM-B  LLM-C  LLM-D   │  ← 4个完整的独立模型
    │  各自独立生成答案，互不知晓       │
    └─────────────────────────────────┘
                    │
            把4个答案拼接 + 原始问题
                    │
    ┌─────────────────────────────────┐
    │         Layer 2（并行）          │
    │  LLM-A  LLM-B  LLM-C  LLM-D   │  ← 同样的模型，但看到了Layer1的输出
    └─────────────────────────────────┘
                    │
    ┌─────────────────────────────────┐
    │         Layer 3（单个）          │
    │              LLM-A              │  ← 最后一层只需一个模型做最终合并
    └─────────────────────────────────┘
                    │
                最终答案
```

**关键特性**：
- **无需训练**：纯 prompt 驱动，即插即用
- **在系统层**：开发者/用户可以完全控制哪些模型参与、分几层
- **模型间独立**：每个 LLM 是完整的独立模型，不共享权重
- **成本叠加**：调用 N 个模型就要付 N 倍的 API 费用

### 三、MoE vs MoA：核心对比

| 维度 | **MoE（专家混合）** | **MoA（Agent 混合）** |
|---|---|---|
| **操作层级** | 模型内部（权重/激活层） | 系统外部（API/prompt 层） |
| **"专家"是什么** | FFN 子网络（几亿参数） | 完整的 LLM（几百亿参数） |
| **门控机制** | 独立的 Gating Network | LLM 自身的理解和综合能力 |
| **是否需要训练** | **必须联合训练** | **完全不需要** |
| **推理成本** | 低（稀疏激活，只用部分参数） | 高（每个模型都要完整推理） |
| **延迟** | 单次推理延迟（毫秒级） | 多轮串行延迟（秒到分钟级） |
| **可扩展性** | 受模型架构限制 | 任意模型即插即用 |
| **用户感知** | 透明（感知不到） | 可见（可以看到多个模型的输出） |
| **适用场景** | 提升单模型能力上限 | 提升系统整体输出质量 |
| **典型代表** | Mixtral、GPT-4、DeepSeek-V3 | Hermes MoA Tool、Together AI |

### 四、论文里的类比解释（最精准的理解）

论文 2.3 节专门做了这个类比，原文说：

> **"MoA 是 MoE 在模型层面的扩展"**

| MoE | MoA |
|---|---|
| 在**激活层**操作 | 在**模型层**操作 |
| 专家 = FFN 子网络 | 专家 = 完整 LLM |
| 门控 = Gating Network | 门控 = LLM 自身的 prompt 理解 |
| 残差连接 = `+ x_i` | 原始问题保留 = `+ x_1` |

**最关键的区别**：MoE 的门控网络是一个独立的小网络，需要训练来学会"把什么问题路由给哪个专家"；而 MoA 直接利用 LLM 本身的语言理解能力来做聚合，不需要额外的门控训练。

---

### 五、一个直觉类比

**MoE** 像一个公司里的**内部分工**：
- 一个员工（模型）内部有多个专业技能模块（专家）
- 接到任务时，大脑（门控网络）自动决定调用哪些技能
- 外部看起来就是一个人在工作

**MoA** 像一个**外部咨询团队**：
- 多个独立顾问（LLM）各自给出意见
- 一个主顾问（Aggregator）综合所有意见出最终报告
- 外部能看到整个协作过程
